本書を研究室で使い始める前に、どうしても伝えておきたいことがある。AIコーディングエージェントの世界は、いま非常に速い速度で変化しているからである。

AIモデルの性能は種類や課金プランによっても異なり、同じモデルに同じ指示を出しても生成されるコードは毎回変わりうる。本書はこの変動を前提とし、特定のモデル出力に依存しない演習設計としている。モデル間の性能差やベンチマーク、再現性への備えについては[§0-7 モデル選択・推論の深さ・コスト意識](./00_ai_agent.md#0-7-モデル選択推論の深さコスト意識)で詳しく述べる。

**情報は 2026年4月時点のものである**。本書で紹介している具体的なツール（Claude Code CLI、Codex CLI など）の仕様、契約プラン、プライバシーポリシー、関連する法規制（個人情報保護法・GDPR・HIPAA など）は、今後変更されうる。特に、入力データがモデルの学習に使われるかどうか、サーバにどのくらいの期間保持されるか、ゼロデータ保持（Zero Data Retention; ZDR）契約が可能かといったプライバシー上の重要項目は、各社が頻繁にポリシーを更新している。本書の記述を最新情報の代わりに使ってはならない。

**制限付きデータ・未発表データの扱いは所属機関に事前相談すること**。ヒトゲノムデータ（dbGaP・JGA・EGA 等）や Data Use Agreement（DUA）の対象データ、査読中の原稿、未公開の新規配列、特許出願前の発明、共同研究者から受け取った未発表素材、NDA・MTA の対象物——これらをクラウド AI エージェントに送信してよいかの判断は、読者個人ではなく**所属機関の情報セキュリティ部門・研究倫理審査委員会・法務部門・産学連携/知財部門**が責任を持つ領域である。本書はその判断を代替しない。本書を参考にしつつも、具体的な運用は必ず所属機関の規程と担当部門の指示に従ってほしい。
