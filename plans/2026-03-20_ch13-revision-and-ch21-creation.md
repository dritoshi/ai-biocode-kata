## A. §21「ソフトウェアの構成要素」新設

### 章構成（~400行、4節）

**21-1. プログラムはどう動くか**（~120行）
- インタプリタ（Python）とコンパイル言語（C/C++）の違い（概要）
- コンパイル→リンク→実行の流れを図解
- 静的リンク vs 動的リンクの比較表
- 共有ライブラリ（.so/.dylib）とLD_LIBRARY_PATH
- ldd/otool -Lコマンドで依存確認
- samtools→libhts→libzの依存チェーン実例
- condaが共有ライブラリを環境内に管理する仕組み
- ※ gccオプション/PIC/dlopen等の詳細は省略

**21-2. Pythonのモジュールシステム**（~120行）
- モジュール（.pyファイル）/パッケージ（ディレクトリ）/ライブラリの定義
- import文とsys.pathの検索順序
- site-packages、PYTHONPATH
- `__init__.py`の役割、相対/絶対import
- よくあるエラー対処（ModuleNotFoundError, ImportError）

**21-3. 依存関係の問題**（~80行）
- 依存チェーン（Python→C→OS）
- 依存地獄（dependency hell）
- venv/condaが隔離するもの・しないもの → §5, §13への橋渡し
- 依存関係の可視化ツール（pipdeptree等）

**21-4. APIとインターフェース**（~80行）
- API概念（CLI / Python / Web の3レベル）
- APIバージョン依存 → §6セマンティックバージョニング参照
- SOLID残り4原則の概要（§1のSRP拡張として）
  - Open/Closed, Interface Segregation: 各1段落+例
  - Liskov, Dependency Inversion: 概要のみ
