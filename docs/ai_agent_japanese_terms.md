# AIコーディングエージェントが日本語会話中に使う用語集

## 日本語混合フレーズ（エージェントの定番表現）

| フレーズ | 意味 |
|---|---|
| 〜にフォールバックします | 失敗時に代替手段に切り替える |
| 〜をエクスポートします | 外部から使えるよう公開する |
| 型安全性を担保する | 型チェックで不正な値を防ぐ |
| 疎結合に保つ | モジュール間の依存を緩くする |
| 関心の分離 | 機能ごとに責務を切り分ける |
| 副作用を避ける | 関数外の状態を変更しない |
| 破壊的変更 | 既存コードが動かなくなる変更（= breaking change） |
| 冪等性を保証する | 何度実行しても同じ結果になる |
| 楽観的ロック / 悲観的ロック | データ競合の制御方式 |
| 早期リターンする | 条件を満たさない場合すぐ関数を抜ける |
| ネストが深い | if/forが何重にも入れ子になっている |
| 循環参照 | モジュールが互いに依存しあう問題 |
| 競合状態（レースコンディション） | 複数処理のタイミングで結果が変わるバグ |
| 非同期で処理する | 完了を待たずに次に進む |
| ストリームで処理する | データを少しずつ流して処理する |
| ツリーシェイキング | 未使用コードをビルド時に除去 |
| ホットリロード | コード変更を即座に反映 |
| デッドコード | 到達しない・使われないコード |
| ユーティリティ関数 | 汎用的な補助関数 |
| ヘルパー関数 | 特定処理を助ける小さな関数 |
| エッジケースを考慮する | 境界条件や例外的入力に対処 |
| ベストプラクティスとしては〜 | 推奨される方法としては〜 |
| トレードオフがあります | 一方を取ると他方を犠牲にする |
| 〜はアンチパターンです | やってはいけない設計 |
| 〜はオーバーキルです | やりすぎ・不必要に大げさ |
| ボトルネックになります | 性能上の律速になる |
| 技術的負債になります | 将来の修正コストが積み上がる |
| 〜をハードコードしないでください | 値を直接埋め込まず変数・設定にする |
| 〜はデプリケート（非推奨）です | 将来削除予定なので使わないほうがいい |
| 〜の責務が大きすぎます | 一つのクラス/関数が多くのことをしすぎ |
| 〜でラップして抽象化します | 詳細を隠して扱いやすくする |
| 環境変数で管理してください | 秘密情報や設定値をコード外に出す |
