## CRITICAL（8件）— 最優先で修正すべき事項

### C-1. §0 / 付録B: Codex CLIの承認モード名が根本的に誤り
- **箇所**: `00_ai_agent.md:59-63行`, `appendix_b_cli_reference.md:27-32行`
- **現状**: Read-Only / Auto（デフォルト）/ `--full-auto` と記載
- **正しくは**: **Suggest**（デフォルト）/ **Auto-Edit** / **Full-Auto** の3段階。`-s read-only` はサンドボックスモードであり承認モードとは別概念
- **根拠**: OpenAI公式ドキュメント (developers.openai.com/codex/agent-approvals-security)

### C-2. §3: `bisect.insort()` の計算量を $O(\log n)$ と記載（正しくは $O(n)$）
- **箇所**: `03_cs_basics.md:173, 189行`
- **現状**: 「検索・挿入ともに $O(\log n)$ で行える」
- **正しくは**: 検索は $O(\log n)$ だが、`insort()` による挿入は $O(n)$（リスト要素のシフトが支配的）
- **根拠**: Python公式ドキュメント bisect モジュール

### C-3. §3: `sum([0.1]*10)` の例がPython 3.12+で再現しない
- **箇所**: `03_cs_basics.md:286-292行`
- **現状**: `sum([0.1]*10)` が `0.9999999999999999` を返すと記載
- **正しくは**: Python 3.12以降ではNeumaier補償加算が導入され `1.0` を返す
- **対応案**: Python 3.12+での変化を注記するか、別の確実に再現する例に差し替え

### C-4. §8: テストファイルパスの章番号不整合（ch07→ch08）
- **箇所**: `08_testing.md:40, 124, 127, 202, 306行`
- **現状**: テストパスが `tests/ch07/` になっている
- **正しくは**: §8のテストなので `tests/ch08/` であるべき

### C-5. §19: GenBankのID形式にRefSeqアクセッション(NM_, NR_, XM_)を記載
- **箇所**: `19_database_api.md:221行`
- **現状**: GenBankの「代表的ID形式」が `NM_, NR_, XM_`
- **正しくは**: `NM_` 等はRefSeqのアクセッション。GenBankは `U12345`, `AF123456` 等のINSDC形式
- **本章70行目でGenBank=一次DB、RefSeq=二次DBと正しく区別しているのにテーブルが矛盾**

### C-6. §21: 冒頭で§20へのリンク欠落＋「本章では」の重複

- **現状**: 前章の振り返りが「本章では」になっており、§20へのリンクもない
- **CLAUDE.md規約違反**: 前章への相互参照リンク必須

### C-7. §21: 節番号が全て「20-X」になっている（正しくは21-X）
