# 各章・付録の文字数集計

## 使い方

プロジェクトルートから以下を実行する。

```bash
python3 scripts/count_chars.py
```

`chapters/` 内の全 Markdown ファイルを走査し、3種類の文字数を Markdown テーブル形式で標準出力に書き出す。

### 依存ライブラリ

- [mistune](https://pypi.org/project/mistune/) — Markdown パーサー（プレーンテキスト抽出に使用）

```bash
pip install mistune
```

### 結果の更新

スクリプトの出力をこのファイル（`docs/char_count.md`）の「結果」セクションに貼り付けて更新する。

## カウント方式

スクリプトは 1 ファイルにつき 3 種類の文字数を算出する。

### 1. 文字数（raw）

```python
len(text)
```

ファイルを UTF-8 で読み込み、Python の `len()` でそのまま数える。Markdown 記法（`#`, `**`, `` ` ``, `|` 等）、コードブロック、空白、改行をすべて含む生の文字数である。

### 2. 本文文字数

Markdown の構造を解析し、読者が実際に目にする本文テキストのみを抽出してからカウントする。

抽出には mistune の `HTMLRenderer` をサブクラス化したカスタムレンダラー `PlainTextRenderer` を使用する。各 Markdown 要素の扱いは以下の通り。

| 要素 | 扱い |
|------|------|
| テキスト・強調(`*`, `**`)・見出し | テキストのみ抽出 |
| リンク `[text](url)` | リンクテキストのみ抽出（URL は除外） |
| インラインコード `` `code` `` | テキストを残す |
| コードブロック ` ```...``` ` | 除外（空文字を返す） |
| テーブル | セル内テキストのみ抽出（`|`, `-` 等の記法は除外） |
| 画像 `![alt](url)` | 除外 |
| HTML・ブロック HTML | 除外 |

抽出されたプレーンテキストに対して `len()` でカウントする。Markdown 記法の分だけ raw 文字数より小さくなる。

### 3. 全角換算

日本語書籍の分量見積もりに適した指標。本文テキスト（上記 2 で抽出したもの）に対し、Unicode の文字幅プロパティを用いて全角文字換算の文字数を算出する。

```python
import unicodedata

def count_zenkaku(text: str) -> int:
    half_widths = 0
    for ch in text:
        eaw = unicodedata.east_asian_width(ch)
        if eaw in ("W", "F"):  # Wide, Fullwidth → 全角
            half_widths += 2
        else:                   # Na, H, N, A → 半角
            half_widths += 1
    return -(-half_widths // 2)  # 端数切り上げ
```

`unicodedata.east_asian_width()` は各 Unicode コードポイントに対して以下のカテゴリを返す（[UAX #11](https://www.unicode.org/reports/tr11/) 準拠）。

| カテゴリ | 略称 | 換算 | 代表的な文字 |
|---------|------|------|------------|
| Wide | W | 1 全角 | ひらがな、カタカナ、漢字、全角記号 |
| Fullwidth | F | 1 全角 | 全角英数字（Ａ, １ 等） |
| Narrow | Na | 0.5 全角 | ASCII 英数字、半角カナ |
| Halfwidth | H | 0.5 全角 | 半角カタカナ |
| Neutral | N | 0.5 全角 | 制御文字等 |
| Ambiguous | A | 0.5 全角 | ギリシャ文字、一部記号（環境依存） |

Ambiguous カテゴリ（ギリシャ文字 α, β や罫線素片 ─ 等）は、CJK 環境では全角幅で表示されることが多いが、ここでは保守的に半角扱いとしている。最終的な半角幅の合計を 2 で割り、端数を切り上げて全角換算文字数とする。

### 指標間の関係

```
文字数（raw） ≥ 本文文字数 ≥ 全角換算
```

- 文字数 → 本文文字数: Markdown 記法・コードブロックの分だけ減る
- 本文文字数 → 全角換算: 半角文字（ASCII 英数字・記号）が 0.5 換算される分だけ減る
